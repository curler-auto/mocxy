# Feature 4.1 — On-Premises Docker Deployment

## Summary

Single-command on-premises deployment of the entire Neuron Interceptor platform via Docker Compose. Customers run `docker compose up -d` and get a fully operational backend with PostgreSQL, Redis, MinIO (S3-compatible storage), the Fastify API server, and Nginx reverse proxy -- all pre-configured and production-ready.

## Why

Enterprise customers require on-premises deployment for data sovereignty, compliance, and air-gapped network requirements. A self-contained Docker Compose stack eliminates the need for complex infrastructure provisioning and allows IT teams to deploy the platform in minutes rather than days. The deployment must be repeatable, upgradable, and include backup/restore capabilities from day one.

## Architecture Context

- **Plugin location**: `health_check/utils/neuron-interceptor-plugin/`
- **Backend**: Fastify API server (from spec 13-backend-api.md)
- **Database**: PostgreSQL 16 for persistent data (rules, workspaces, users, audit logs)
- **Cache**: Redis 7 for session management, rate limiting, and pub/sub
- **Object storage**: MinIO for mock body storage and data exports
- **Reverse proxy**: Nginx for SSL termination, static file serving, WebSocket support
- **Deployment directory**: `health_check/utils/neuron-interceptor-plugin/deploy/onprem/`

## Directory Structure

```
deploy/onprem/
  docker-compose.yml        # Service orchestration
  .env.example              # Template for environment variables
  nginx.conf                # Reverse proxy configuration
  Dockerfile                # Multi-stage build for the API service
  init.sh                   # First-run setup script
  backup.sh                 # Backup all data
  restore.sh                # Restore from backup archive
  upgrade.sh                # Pull latest images, migrate, restart
  README.md                 # Installation and operations guide
```

### File 2: `deploy/onprem/Dockerfile`

Multi-stage build for the Fastify API service. Stage 1 installs dependencies and compiles. Stage 2 copies only production artifacts into a minimal runtime image.

```dockerfile
# =============================================================================
# Neuron Interceptor — API Server Dockerfile
# Multi-stage build: Node 20 Alpine build → Node 20 Alpine runtime
# =============================================================================

# --- Stage 1: Build ----------------------------------------------------------
FROM node:20-alpine AS builder

WORKDIR /build

# Install build dependencies for native modules (bcrypt, pg-native, etc.)
RUN apk add --no-cache python3 make g++ linux-headers

# Copy package manifests first for layer caching
COPY server/package.json server/package-lock.json ./

# Install ALL dependencies (including devDependencies for build)
RUN npm ci

# Copy source code
COPY server/ ./

# Prune devDependencies for production
RUN npm prune --production

# --- Stage 2: Runtime --------------------------------------------------------
FROM node:20-alpine AS runtime

# Security: run as non-root user
RUN addgroup -S neuron && adduser -S neuron -G neuron

WORKDIR /app

# Install runtime system dependencies
RUN apk add --no-cache tini curl

# Copy production node_modules and source from builder
COPY --from=builder --chown=neuron:neuron /build/node_modules ./node_modules
COPY --from=builder --chown=neuron:neuron /build/src ./src
COPY --from=builder --chown=neuron:neuron /build/package.json ./package.json
COPY --from=builder --chown=neuron:neuron /build/knexfile.js ./knexfile.js
COPY --from=builder --chown=neuron:neuron /build/migrations ./migrations

# Create directories for data
RUN mkdir -p /app/data /app/logs && chown -R neuron:neuron /app/data /app/logs

# Switch to non-root user
USER neuron

# Expose API port
EXPOSE 3001

# Health check
HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
  CMD curl -f http://localhost:3001/health || exit 1

# Use tini as init process for proper signal handling
ENTRYPOINT ["tini", "--"]

# Start the server
CMD ["node", "src/index.js"]
```

### File 3: `deploy/onprem/nginx.conf`

Full Nginx reverse proxy configuration with WebSocket upgrade support, CORS headers, gzip compression, SSL termination, security headers, and static file serving.

```nginx
# =============================================================================
# Neuron Interceptor — Nginx Reverse Proxy Configuration
# =============================================================================

worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
    multi_accept on;
}

http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    # --- Logging -------------------------------------------------------------
    log_format main '$remote_addr - $remote_user [$time_local] '
                    '"$request" $status $body_bytes_sent '
                    '"$http_referer" "$http_user_agent" '
                    'rt=$request_time';

    access_log /var/log/nginx/access.log main;

    # --- Performance ---------------------------------------------------------
    sendfile        on;
    tcp_nopush      on;
    tcp_nodelay     on;
    keepalive_timeout 65;
    types_hash_max_size 2048;
    client_max_body_size 50m;

    # --- Gzip Compression ----------------------------------------------------
    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_min_length 256;
    gzip_types
        text/plain
        text/css
        text/xml
        text/javascript
        application/json
        application/javascript
        application/xml
        application/rss+xml
        application/atom+xml
        image/svg+xml
        font/woff2;

    # --- Rate Limiting -------------------------------------------------------
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=30r/s;
    limit_req_zone $binary_remote_addr zone=auth_limit:10m rate=5r/s;

    # --- Upstream Definitions ------------------------------------------------
    upstream api_backend {
        server api:3001;
        keepalive 32;
    }

    # --- WebSocket Upgrade Map -----------------------------------------------
    map $http_upgrade $connection_upgrade {
        default upgrade;
        ''      close;
    }

    # --- HTTP → HTTPS Redirect -----------------------------------------------
    server {
        listen 80;
        server_name _;

        # Let's Encrypt ACME challenge support
        location /.well-known/acme-challenge/ {
            root /var/www/certbot;
        }

        location / {
            return 301 https://$host$request_uri;
        }
    }

    # --- Main HTTPS Server ---------------------------------------------------
    server {
        listen 443 ssl http2;
        server_name _;

        # --- SSL Configuration -----------------------------------------------
        ssl_certificate     /etc/nginx/certs/server.crt;
        ssl_certificate_key /etc/nginx/certs/server.key;
        ssl_protocols       TLSv1.2 TLSv1.3;
        ssl_ciphers         HIGH:!aNULL:!MD5:!RC4;
        ssl_prefer_server_ciphers on;
        ssl_session_cache   shared:SSL:10m;
        ssl_session_timeout 1d;
        ssl_session_tickets off;

        # --- Security Headers ------------------------------------------------
        add_header X-Frame-Options "SAMEORIGIN" always;
        add_header X-Content-Type-Options "nosniff" always;
        add_header X-XSS-Protection "1; mode=block" always;
        add_header Referrer-Policy "strict-origin-when-cross-origin" always;
        add_header Strict-Transport-Security "max-age=63072000; includeSubDomains" always;

        # --- CORS Headers (applied globally) ----------------------------------
        add_header Access-Control-Allow-Origin "$http_origin" always;
        add_header Access-Control-Allow-Methods "GET, POST, PUT, PATCH, DELETE, OPTIONS" always;
        add_header Access-Control-Allow-Headers "Authorization, Content-Type, X-Requested-With, X-Workspace-Id" always;
        add_header Access-Control-Allow-Credentials "true" always;
        add_header Access-Control-Max-Age "86400" always;

        # Handle preflight OPTIONS requests
        if ($request_method = 'OPTIONS') {
            return 204;
        }

        # --- Static Landing Page ----------------------------------------------
        location / {
            root /usr/share/nginx/html;
            index index.html;
            try_files $uri $uri/ /index.html;

            # Cache static assets aggressively
            location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
                expires 30d;
                add_header Cache-Control "public, immutable";
            }
        }

        # --- API Proxy --------------------------------------------------------
        location /api/ {
            limit_req zone=api_limit burst=50 nodelay;

            proxy_pass http://api_backend/api/;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_set_header Connection "";

            # Timeouts
            proxy_connect_timeout 30s;
            proxy_send_timeout 60s;
            proxy_read_timeout 60s;

            # Buffering
            proxy_buffering on;
            proxy_buffer_size 4k;
            proxy_buffers 8 16k;
        }

        # --- Auth Routes (stricter rate limit) --------------------------------
        location /api/auth/ {
            limit_req zone=auth_limit burst=10 nodelay;

            proxy_pass http://api_backend/api/auth/;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_set_header Connection "";
        }

        # --- WebSocket Proxy --------------------------------------------------
        location /ws {
            proxy_pass http://api_backend/ws;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection $connection_upgrade;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;

            # WebSocket-specific timeouts (longer for persistent connections)
            proxy_connect_timeout 7d;
            proxy_send_timeout 7d;
            proxy_read_timeout 7d;
        }

        # --- Health Check (no rate limit) -------------------------------------
        location /health {
            proxy_pass http://api_backend/health;
            proxy_http_version 1.1;
            proxy_set_header Connection "";
            access_log off;
        }

        # --- MinIO Console (admin access only) --------------------------------
        location /minio/ {
            # Restrict to internal networks in production
            # allow 10.0.0.0/8;
            # allow 172.16.0.0/12;
            # allow 192.168.0.0/16;
            # deny all;

            proxy_pass http://minio:9001/;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection $connection_upgrade;
        }
    }
}
```

### File 4: `deploy/onprem/docker-compose.yml`

Orchestrates all 5 services with health checks, dependency ordering, named volumes, and network isolation.

```yaml
# =============================================================================
# Neuron Interceptor — On-Premises Docker Compose
# Usage: docker compose up -d
# First run: ./init.sh && docker compose up -d
# =============================================================================

version: "3.9"

services:
  # ---------------------------------------------------------------------------
  # PostgreSQL 16 — Primary data store
  # ---------------------------------------------------------------------------
  postgres:
    image: postgres:16-alpine
    container_name: neuron-postgres
    restart: unless-stopped
    environment:
      POSTGRES_DB: ${POSTGRES_DB:-neuron}
      POSTGRES_USER: ${POSTGRES_USER:-neuron}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:?POSTGRES_PASSWORD is required}
      PGDATA: /var/lib/postgresql/data/pgdata
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "${POSTGRES_PORT:-5432}:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER:-neuron} -d ${POSTGRES_DB:-neuron}"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s
    networks:
      - neuron-internal

  # ---------------------------------------------------------------------------
  # Redis 7 — Session cache, pub/sub, rate limiting
  # ---------------------------------------------------------------------------
  redis:
    image: redis:7-alpine
    container_name: neuron-redis
    restart: unless-stopped
    command: >
      redis-server
      --requirepass ${REDIS_PASSWORD:?REDIS_PASSWORD is required}
      --appendonly yes
      --appendfsync everysec
      --maxmemory 256mb
      --maxmemory-policy allkeys-lru
    volumes:
      - redis_data:/data
    ports:
      - "${REDIS_PORT:-6379}:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "${REDIS_PASSWORD}", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 10s
    networks:
      - neuron-internal

  # ---------------------------------------------------------------------------
  # MinIO — S3-compatible object storage for mock bodies and exports
  # ---------------------------------------------------------------------------
  minio:
    image: minio/minio:latest
    container_name: neuron-minio
    restart: unless-stopped
    command: server /data --console-address ":9001"
    environment:
      MINIO_ROOT_USER: ${MINIO_ROOT_USER:-neuron-minio}
      MINIO_ROOT_PASSWORD: ${MINIO_ROOT_PASSWORD:?MINIO_ROOT_PASSWORD is required}
    volumes:
      - minio_data:/data
    ports:
      - "${MINIO_PORT:-9000}:9000"
      - "${MINIO_CONSOLE_PORT:-9001}:9001"
    healthcheck:
      test: ["CMD", "mc", "ready", "local"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 15s
    networks:
      - neuron-internal

  # ---------------------------------------------------------------------------
  # API — Fastify backend server
  # ---------------------------------------------------------------------------
  api:
    build:
      context: ../../server
      dockerfile: ../deploy/onprem/Dockerfile
    container_name: neuron-api
    restart: unless-stopped
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
      minio:
        condition: service_healthy
    environment:
      NODE_ENV: ${NODE_ENV:-production}
      PORT: ${API_PORT:-3001}
      LOG_LEVEL: ${LOG_LEVEL:-info}

      # Database
      DATABASE_URL: postgresql://${POSTGRES_USER:-neuron}:${POSTGRES_PASSWORD}@postgres:5432/${POSTGRES_DB:-neuron}

      # Redis
      REDIS_URL: redis://:${REDIS_PASSWORD}@redis:6379

      # MinIO / S3
      S3_ENDPOINT: http://minio:9000
      S3_ACCESS_KEY: ${MINIO_ROOT_USER:-neuron-minio}
      S3_SECRET_KEY: ${MINIO_ROOT_PASSWORD}
      S3_BUCKET: ${MINIO_BUCKET:-neuron-data}
      S3_FORCE_PATH_STYLE: "true"

      # Auth
      JWT_SECRET: ${JWT_SECRET:?JWT_SECRET is required}
      SESSION_SECRET: ${SESSION_SECRET:?SESSION_SECRET is required}

      # License
      LICENSE_FILE_PATH: /app/license/license.lic
    volumes:
      - ./license.lic:/app/license/license.lic:ro
      - api_logs:/app/logs
    ports:
      - "${API_PORT:-3001}:3001"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3001/health"]
      interval: 15s
      timeout: 5s
      retries: 3
      start_period: 30s
    networks:
      - neuron-internal

  # ---------------------------------------------------------------------------
  # Nginx — Reverse proxy, SSL termination, static files
  # ---------------------------------------------------------------------------
  nginx:
    image: nginx:1.25-alpine
    container_name: neuron-nginx
    restart: unless-stopped
    depends_on:
      api:
        condition: service_healthy
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - ./certs:/etc/nginx/certs:ro
      - ./landing-page:/usr/share/nginx/html:ro
      - nginx_logs:/var/log/nginx
    ports:
      - "${NGINX_HTTP_PORT:-80}:80"
      - "${NGINX_HTTPS_PORT:-443}:443"
    healthcheck:
      test: ["CMD", "wget", "--no-verbose", "--tries=1", "--spider", "http://localhost:80/"]
      interval: 15s
      timeout: 5s
      retries: 3
      start_period: 10s
    networks:
      - neuron-internal

# =============================================================================
# Volumes — Persistent data survives container restarts and rebuilds
# =============================================================================
volumes:
  postgres_data:
    driver: local
  redis_data:
    driver: local
  minio_data:
    driver: local
  api_logs:
    driver: local
  nginx_logs:
    driver: local

# =============================================================================
# Networks — Internal network isolates services from host
# =============================================================================
networks:
  neuron-internal:
    driver: bridge
    name: neuron-internal
```

### File 5: `deploy/onprem/init.sh`

First-run script that generates random secrets, creates the `.env` file, generates self-signed SSL certificates, creates the admin user, and runs database migrations.

```bash
#!/usr/bin/env bash
# =============================================================================
# Neuron Interceptor — First-Run Initialization
# Generates secrets, creates .env, sets up SSL, runs migrations, creates admin.
# Usage: ./init.sh
# =============================================================================

set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
cd "$SCRIPT_DIR"

# --- Colors for output -------------------------------------------------------
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
CYAN='\033[0;36m'
NC='\033[0m' # No Color

log()  { echo -e "${GREEN}[INIT]${NC} $*"; }
warn() { echo -e "${YELLOW}[WARN]${NC} $*"; }
err()  { echo -e "${RED}[ERROR]${NC} $*" >&2; }

# --- Helper: generate a random secret ----------------------------------------
generate_secret() {
  openssl rand -base64 "$1" 2>/dev/null | tr -d '\n/+=' | head -c "$1"
}

# --- Step 1: Generate .env if it doesn't exist --------------------------------
if [ ! -f .env ]; then
  log "No .env file found. Generating from .env.example..."

  if [ ! -f .env.example ]; then
    err ".env.example not found. Cannot generate .env."
    exit 1
  fi

  cp .env.example .env

  # Generate random secrets
  JWT_SECRET=$(generate_secret 48)
  SESSION_SECRET=$(generate_secret 48)
  POSTGRES_PASSWORD=$(generate_secret 32)
  REDIS_PASSWORD=$(generate_secret 32)
  MINIO_ROOT_PASSWORD=$(generate_secret 32)
  ADMIN_PASSWORD=$(generate_secret 16)

  # Replace empty values in .env
  if [[ "$OSTYPE" == "darwin"* ]]; then
    SED_INPLACE=(-i '')
  else
    SED_INPLACE=(-i)
  fi

  sed "${SED_INPLACE[@]}" "s|^JWT_SECRET=.*|JWT_SECRET=${JWT_SECRET}|" .env
  sed "${SED_INPLACE[@]}" "s|^SESSION_SECRET=.*|SESSION_SECRET=${SESSION_SECRET}|" .env
  sed "${SED_INPLACE[@]}" "s|^POSTGRES_PASSWORD=.*|POSTGRES_PASSWORD=${POSTGRES_PASSWORD}|" .env
  sed "${SED_INPLACE[@]}" "s|^REDIS_PASSWORD=.*|REDIS_PASSWORD=${REDIS_PASSWORD}|" .env
  sed "${SED_INPLACE[@]}" "s|^MINIO_ROOT_PASSWORD=.*|MINIO_ROOT_PASSWORD=${MINIO_ROOT_PASSWORD}|" .env
  sed "${SED_INPLACE[@]}" "s|^ADMIN_PASSWORD=.*|ADMIN_PASSWORD=${ADMIN_PASSWORD}|" .env

  log "Generated .env with random secrets."
  warn "IMPORTANT: Save the admin password: ${ADMIN_PASSWORD}"
  warn "IMPORTANT: Review and customize .env before proceeding."
else
  log ".env already exists. Skipping secret generation."
fi

# Source the .env file
set -a
source .env
set +a

# --- Step 2: Generate self-signed SSL certificates ----------------------------
CERT_DIR="$SCRIPT_DIR/certs"
if [ ! -f "$CERT_DIR/server.crt" ] || [ ! -f "$CERT_DIR/server.key" ]; then
  log "Generating self-signed SSL certificates..."
  mkdir -p "$CERT_DIR"

  openssl req -x509 -nodes -days 365 \
    -newkey rsa:2048 \
    -keyout "$CERT_DIR/server.key" \
    -out "$CERT_DIR/server.crt" \
    -subj "/C=US/ST=State/L=City/O=NeuronInterceptor/CN=${DOMAIN:-localhost}" \
    -addext "subjectAltName=DNS:${DOMAIN:-localhost},DNS:localhost,IP:127.0.0.1" \
    2>/dev/null

  chmod 600 "$CERT_DIR/server.key"
  chmod 644 "$CERT_DIR/server.crt"
  log "SSL certificates generated in $CERT_DIR/"
  warn "For production, replace with real certificates from your CA."
else
  log "SSL certificates already exist. Skipping generation."
fi

# --- Step 3: Create required directories --------------------------------------
mkdir -p "$SCRIPT_DIR/backups"
mkdir -p "$SCRIPT_DIR/landing-page"

# Create a placeholder landing page if none exists
if [ ! -f "$SCRIPT_DIR/landing-page/index.html" ]; then
  cat > "$SCRIPT_DIR/landing-page/index.html" <<'LANDING_EOF'
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Neuron Interceptor</title>
  <style>
    body { margin: 0; background: #1e1e2e; color: #cdd6f4; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif; display: flex; align-items: center; justify-content: center; min-height: 100vh; }
    .container { text-align: center; max-width: 600px; padding: 2rem; }
    h1 { color: #89b4fa; font-size: 2.5rem; margin-bottom: 0.5rem; }
    p { color: #a6adc8; font-size: 1.1rem; line-height: 1.6; }
    .status { display: inline-block; padding: 0.5rem 1.5rem; background: #313244; border-radius: 8px; margin-top: 1rem; color: #a6e3a1; font-weight: 600; }
  </style>
</head>
<body>
  <div class="container">
    <h1>Neuron Interceptor</h1>
    <p>Enterprise API interception, mocking, and debugging platform.</p>
    <div class="status">Platform Running</div>
  </div>
</body>
</html>
LANDING_EOF
  log "Created placeholder landing page."
fi

# --- Step 4: Create empty license placeholder ---------------------------------
if [ ! -f "$SCRIPT_DIR/license.lic" ]; then
  touch "$SCRIPT_DIR/license.lic"
  log "Created empty license.lic placeholder. Add your license file before starting."
fi

# --- Step 5: Start database and run migrations --------------------------------
log "Starting PostgreSQL for migrations..."
docker compose up -d postgres
log "Waiting for PostgreSQL to be healthy..."

RETRIES=30
until docker compose exec postgres pg_isready -U "${POSTGRES_USER:-neuron}" -d "${POSTGRES_DB:-neuron}" > /dev/null 2>&1; do
  RETRIES=$((RETRIES - 1))
  if [ "$RETRIES" -le 0 ]; then
    err "PostgreSQL failed to become healthy after 30 attempts."
    exit 1
  fi
  sleep 2
done
log "PostgreSQL is ready."

# Run database migrations via the API container
log "Running database migrations..."
docker compose run --rm --no-deps api npx knex migrate:latest
log "Migrations completed."

# --- Step 6: Create admin user ------------------------------------------------
log "Creating admin user..."
docker compose run --rm --no-deps api node -e "
```

### File 6: `deploy/onprem/backup.sh`

Backs up PostgreSQL (pg_dump), Redis (SAVE + copy AOF), and creates a timestamped tar archive.

```bash
#!/usr/bin/env bash
# =============================================================================
# Neuron Interceptor — Backup Script
# Creates a timestamped backup of PostgreSQL, Redis, and configuration.
# Usage: ./backup.sh
# =============================================================================

set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
cd "$SCRIPT_DIR"

# --- Colors -------------------------------------------------------------------
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m'

log()  { echo -e "${GREEN}[BACKUP]${NC} $*"; }
warn() { echo -e "${YELLOW}[BACKUP]${NC} $*"; }
err()  { echo -e "${RED}[BACKUP]${NC} $*" >&2; }

# --- Load environment ---------------------------------------------------------
if [ ! -f .env ]; then
  err ".env not found. Run init.sh first."
  exit 1
fi

set -a
source .env
set +a

BACKUP_DIR="${BACKUP_DIR:-./backups}"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_NAME="neuron-backup-${TIMESTAMP}"
WORK_DIR="${BACKUP_DIR}/${BACKUP_NAME}"

mkdir -p "$WORK_DIR"

# --- Step 1: PostgreSQL dump --------------------------------------------------
log "Backing up PostgreSQL..."
docker compose exec -T postgres pg_dump \
  -U "${POSTGRES_USER:-neuron}" \
  -d "${POSTGRES_DB:-neuron}" \
  --format=custom \
  --compress=9 \
  > "${WORK_DIR}/postgres.dump"

PG_SIZE=$(du -h "${WORK_DIR}/postgres.dump" | cut -f1)
log "PostgreSQL backup: ${PG_SIZE}"

# --- Step 2: Redis SAVE + copy ------------------------------------------------
log "Backing up Redis..."
docker compose exec -T redis redis-cli -a "${REDIS_PASSWORD}" SAVE > /dev/null 2>&1

# Copy the RDB file from the container
docker compose cp redis:/data/dump.rdb "${WORK_DIR}/redis-dump.rdb" 2>/dev/null || warn "Redis RDB copy failed (may not exist yet)."

REDIS_SIZE=$(du -h "${WORK_DIR}/redis-dump.rdb" 2>/dev/null | cut -f1 || echo "0B")
log "Redis backup: ${REDIS_SIZE}"

# --- Step 3: Copy configuration files -----------------------------------------
log "Backing up configuration..."
cp .env "${WORK_DIR}/env.bak"
cp nginx.conf "${WORK_DIR}/nginx.conf.bak"
cp docker-compose.yml "${WORK_DIR}/docker-compose.yml.bak"

if [ -f license.lic ]; then
  cp license.lic "${WORK_DIR}/license.lic.bak"
fi

# --- Step 4: Create tar archive -----------------------------------------------
log "Creating archive..."
tar -czf "${BACKUP_DIR}/${BACKUP_NAME}.tar.gz" -C "${BACKUP_DIR}" "${BACKUP_NAME}"
rm -rf "$WORK_DIR"

ARCHIVE_SIZE=$(du -h "${BACKUP_DIR}/${BACKUP_NAME}.tar.gz" | cut -f1)
log "Archive created: ${BACKUP_DIR}/${BACKUP_NAME}.tar.gz (${ARCHIVE_SIZE})"

# --- Step 5: Prune old backups -----------------------------------------------
RETENTION_DAYS="${BACKUP_RETENTION_DAYS:-30}"
PRUNED=$(find "$BACKUP_DIR" -name "neuron-backup-*.tar.gz" -mtime +"$RETENTION_DAYS" -delete -print | wc -l | tr -d ' ')
if [ "$PRUNED" -gt 0 ]; then
  log "Pruned ${PRUNED} backup(s) older than ${RETENTION_DAYS} days."
fi

# --- Done ---------------------------------------------------------------------
echo ""
log "============================================"
log "  Backup complete!"
log "============================================"
log "  Archive: ${BACKUP_DIR}/${BACKUP_NAME}.tar.gz"
log "  Size:    ${ARCHIVE_SIZE}"
echo ""
```

### File 7: `deploy/onprem/restore.sh`

Restores from a backup archive: stops services, restores PostgreSQL, restores Redis, restarts.

```bash
#!/usr/bin/env bash
# =============================================================================
# Neuron Interceptor — Restore Script
# Restores PostgreSQL, Redis, and config from a backup archive.
# Usage: ./restore.sh <path-to-backup-archive.tar.gz>
# =============================================================================

set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
cd "$SCRIPT_DIR"

# --- Colors -------------------------------------------------------------------
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m'

log()  { echo -e "${GREEN}[RESTORE]${NC} $*"; }
warn() { echo -e "${YELLOW}[RESTORE]${NC} $*"; }
err()  { echo -e "${RED}[RESTORE]${NC} $*" >&2; }

# --- Validate arguments -------------------------------------------------------
if [ $# -lt 1 ]; then
  err "Usage: ./restore.sh <path-to-backup-archive.tar.gz>"
  exit 1
fi

ARCHIVE="$1"
if [ ! -f "$ARCHIVE" ]; then
  err "Archive not found: $ARCHIVE"
  exit 1
fi

# --- Load environment ---------------------------------------------------------
if [ ! -f .env ]; then
  err ".env not found. Run init.sh first or restore .env manually."
  exit 1
fi

set -a
source .env
set +a

# --- Confirmation prompt ------------------------------------------------------
echo ""
warn "This will OVERWRITE the current database and Redis data."
warn "Archive: $ARCHIVE"
read -p "Are you sure you want to proceed? (yes/no): " CONFIRM
if [ "$CONFIRM" != "yes" ]; then
  log "Restore cancelled."
  exit 0
fi

# --- Step 1: Extract archive --------------------------------------------------
log "Extracting archive..."
TEMP_DIR=$(mktemp -d)
tar -xzf "$ARCHIVE" -C "$TEMP_DIR"

# Find the extracted directory (should be neuron-backup-YYYYMMDD_HHMMSS/)
RESTORE_DIR=$(find "$TEMP_DIR" -maxdepth 1 -type d -name "neuron-backup-*" | head -1)
if [ -z "$RESTORE_DIR" ]; then
  err "Invalid archive structure. Expected neuron-backup-* directory."
  rm -rf "$TEMP_DIR"
  exit 1
fi

log "Extracted to: $RESTORE_DIR"

# --- Step 2: Stop API and Nginx (keep DB and Redis running) -------------------
log "Stopping API and Nginx..."
docker compose stop api nginx 2>/dev/null || true

# --- Step 3: Restore PostgreSQL -----------------------------------------------
if [ -f "${RESTORE_DIR}/postgres.dump" ]; then
  log "Restoring PostgreSQL..."

  # Ensure PostgreSQL is running
  docker compose up -d postgres
  sleep 5

  # Drop and recreate database
  docker compose exec -T postgres psql -U "${POSTGRES_USER:-neuron}" -c \
    "SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE datname = '${POSTGRES_DB:-neuron}' AND pid <> pg_backend_pid();" \
    postgres > /dev/null 2>&1 || true

  docker compose exec -T postgres dropdb -U "${POSTGRES_USER:-neuron}" --if-exists "${POSTGRES_DB:-neuron}"
  docker compose exec -T postgres createdb -U "${POSTGRES_USER:-neuron}" "${POSTGRES_DB:-neuron}"

  # Restore from dump
  cat "${RESTORE_DIR}/postgres.dump" | docker compose exec -T postgres pg_restore \
    -U "${POSTGRES_USER:-neuron}" \
    -d "${POSTGRES_DB:-neuron}" \
    --no-owner \
    --no-privileges \
    --if-exists \
    --clean

  log "PostgreSQL restored."
else
  warn "No PostgreSQL dump found in archive. Skipping."
fi

# --- Step 4: Restore Redis ----------------------------------------------------
if [ -f "${RESTORE_DIR}/redis-dump.rdb" ]; then
  log "Restoring Redis..."

  docker compose stop redis
  docker compose cp "${RESTORE_DIR}/redis-dump.rdb" redis:/data/dump.rdb
  docker compose start redis
  sleep 3

  log "Redis restored."
else
  warn "No Redis dump found in archive. Skipping."
fi

# --- Step 5: Restore configuration (optional) ---------------------------------
if [ -f "${RESTORE_DIR}/env.bak" ]; then
  warn "Backup contains .env file. Current .env was NOT overwritten."
  warn "To restore config: cp ${RESTORE_DIR}/env.bak .env"
fi

if [ -f "${RESTORE_DIR}/license.lic.bak" ]; then
  cp "${RESTORE_DIR}/license.lic.bak" "$SCRIPT_DIR/license.lic"
  log "License file restored."
fi

# --- Step 6: Restart all services ---------------------------------------------
log "Restarting all services..."
docker compose up -d

# --- Cleanup ------------------------------------------------------------------
rm -rf "$TEMP_DIR"

# --- Done ---------------------------------------------------------------------
echo ""
log "============================================"
log "  Restore complete!"
log "============================================"
log "  Source: $ARCHIVE"
log "  All services are starting. Check status with:"
log "    docker compose ps"
log "    docker compose logs -f api"
echo ""
```

### File 8: `deploy/onprem/upgrade.sh`

Pulls latest images, runs migrations, and restarts services with zero-downtime intent.

```bash
#!/usr/bin/env bash
# =============================================================================
# Neuron Interceptor — Upgrade Script
# Pulls latest images, runs migrations, restarts services.
# Usage: ./upgrade.sh
# =============================================================================

set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
cd "$SCRIPT_DIR"

# --- Colors -------------------------------------------------------------------
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m'

log()  { echo -e "${GREEN}[UPGRADE]${NC} $*"; }
warn() { echo -e "${YELLOW}[UPGRADE]${NC} $*"; }
err()  { echo -e "${RED}[UPGRADE]${NC} $*" >&2; }

# --- Load environment ---------------------------------------------------------
if [ ! -f .env ]; then
  err ".env not found. Run init.sh first."
  exit 1
fi

set -a
source .env
set +a

# --- Step 1: Pre-flight checks -----------------------------------------------
log "Running pre-flight checks..."

if ! docker compose ps --services | grep -q "api"; then
  err "Services are not running. Start with: docker compose up -d"
  exit 1
fi

# Check current health
HEALTH=$(docker compose exec -T api curl -sf http://localhost:3001/health 2>/dev/null || echo '{"status":"unavailable"}')
log "Current health: $HEALTH"

# --- Step 2: Create backup before upgrade -------------------------------------
log "Creating pre-upgrade backup..."
if [ -f backup.sh ]; then
  bash backup.sh
else
  warn "backup.sh not found. Skipping pre-upgrade backup."
fi

# --- Step 3: Pull latest images -----------------------------------------------
log "Pulling latest images..."
docker compose pull postgres redis minio nginx

# Rebuild the API image with latest code
log "Rebuilding API image..."
docker compose build --no-cache api

# --- Step 4: Run database migrations -----------------------------------------
log "Running database migrations..."
docker compose run --rm --no-deps api npx knex migrate:latest
log "Migrations completed."

# --- Step 5: Restart services (rolling) ---------------------------------------
log "Restarting services..."

# Restart infrastructure services first
docker compose up -d postgres redis minio
log "Waiting for infrastructure services..."
sleep 10

# Restart API
docker compose up -d api
log "Waiting for API to become healthy..."

RETRIES=30
until docker compose exec -T api curl -sf http://localhost:3001/health > /dev/null 2>&1; do
  RETRIES=$((RETRIES - 1))
  if [ "$RETRIES" -le 0 ]; then
    err "API failed to become healthy after upgrade."
    err "Rolling back: docker compose down && docker compose up -d"
    exit 1
  fi
  sleep 2
done
log "API is healthy."

# Restart Nginx last
docker compose up -d nginx
log "Nginx restarted."

# --- Step 6: Post-upgrade health check ---------------------------------------
log "Running post-upgrade health check..."
sleep 3

HEALTH=$(docker compose exec -T api curl -sf http://localhost:3001/health 2>/dev/null || echo '{"status":"unavailable"}')
log "Post-upgrade health: $HEALTH"

# --- Step 7: Prune old Docker images -----------------------------------------
log "Pruning unused Docker images..."
docker image prune -f > /dev/null 2>&1

# --- Done ---------------------------------------------------------------------
echo ""
log "============================================"
log "  Upgrade complete!"
log "============================================"
log "  Health: $HEALTH"
log "  Check status: docker compose ps"
log "  View logs:    docker compose logs -f api"
echo ""
```

### Health Check Endpoint

The API service must expose a `GET /health` endpoint that reports the status of all dependencies. This endpoint is used by Docker health checks, the Nginx proxy, and the upgrade script.

Add to `server/src/routes/health.js`:

```javascript
'use strict';

/**
 * Health check route — reports server status and dependency connectivity.
 * GET /health → { status, version, uptime, db, redis, minio }
 */
async function healthRoutes(fastify) {
  fastify.get('/health', async (request, reply) => {
    const result = {
      status: 'ok',
      version: process.env.npm_package_version || '1.0.0',
      uptime: Math.floor(process.uptime()),
      timestamp: new Date().toISOString(),
      db: 'disconnected',
      redis: 'disconnected',
      minio: 'disconnected',
    };

    // Check PostgreSQL
    try {
      await fastify.knex.raw('SELECT 1');
      result.db = 'connected';
    } catch (err) {
      result.status = 'degraded';
      result.dbError = err.message;
    }

    // Check Redis
    try {
      const pong = await fastify.redis.ping();
      result.redis = pong === 'PONG' ? 'connected' : 'error';
    } catch (err) {
      result.status = 'degraded';
      result.redisError = err.message;
    }

    // Check MinIO
    try {
      await fastify.s3.listBuckets();
      result.minio = 'connected';
    } catch (err) {
      result.status = 'degraded';
      result.minioError = err.message;
    }

    const statusCode = result.status === 'ok' ? 200 : 503;
    return reply.code(statusCode).send(result);
  });
}

module.exports = healthRoutes;
```
